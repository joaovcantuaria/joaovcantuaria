PASSO 1: ABRIR A PÁGINA DE LOGIN
--------------------------------
1. Abra o arquivo "login.html" no navegador
2. Você verá a página de login elegante com design em duas colunas

PASSO 2: FAZER LOGIN (SIMULADO)
--------------------------------
⚠️ IMPORTANTE: O sistema está em MODO SIMULAÇÃO para testes.
Qualquer e-mail e senha funcionarão!

Exemplo de teste:

- Senha: 123456 (mínimo 6 caracteres)

3. Clique em "Entrar"
4. Você será redirecionado para "area-cliente.html"

PASSO 3: EXPLORAR O DASHBOARD
--------------------------------
Após o login, você verá:

✅ SIDEBAR (Menu Lateral Esquerdo):
   - Logo da empresa
   - Menu com 6 seções:
     • Dashboard
     • Meus Projetos
     • Galeria
     • Mensagens (com badge de 3 novas)
     • Arquivos
     • Meu Perfil
   - Botão de Logout (vermelho)

✅ TOPBAR (Barra Superior):
   - Botão de menu mobile (apenas em telas pequenas)
   - Título da página atual
   - Ícone de notificações (com badge de 5)
   - Avatar e nome do usuário

✅ CONTEÚDO PRINCIPAL:
   - Card de boas-vindas
   - 4 cards de estatísticas:
     • 3 Projetos Ativos
     • 12 Projetos Concluídos
     • 247 Fotos Entregues
     • 3 Mensagens Novas
   - Lista de projetos recentes

PASSO 4: TESTAR NAVEGAÇÃO
--------------------------------
Clique nos itens do menu lateral:

1. Dashboard: Visão geral (página inicial)
2. Meus Projetos: Lista de projetos com barra de progresso
3. Galeria: Grid de fotos com overlay para visualizar/baixar
4. Mensagens: (em desenvolvimento)
5. Arquivos: (em desenvolvimento)
6. Meu Perfil: (em desenvolvimento)

PASSO 5: TESTAR FUNCIONALIDADES
--------------------------------

✅ Galeria de Fotos:
   - Passe o mouse sobre uma foto
   - Clique no ícone de olho (👁️) para visualizar em tela cheia
   - Clique no ícone de download (⬇️) para baixar

✅ Menu do Usuário:
   - Clique no avatar/nome do usuário no canto superior direito
   - Verá um dropdown com opções

✅ Notificações:
   - Clique no sino de notificações
   - Verá mensagem informativa

✅ Logout:
   - Clique no botão "Sair" na sidebar
   - Confirme a ação
   - Será redirecionado para login.html

PASSO 6: TESTAR CADASTRO
--------------------------------
1. Na página de login, clique em "Cadastre-se aqui"
2. Preencha o formulário:
   - Nome e Sobrenome
   - E-mail
   - Telefone
   - Empresa (opcional)
   - Senha e Confirmação
3. Aceite os termos
4. Clique em "Criar Conta"
5. Será redirecionado para o login

PASSO 7: TESTAR RESPONSIVIDADE
--------------------------------
Redimensione a janela do navegador ou:
1. Pressione F12 (DevTools)
2. Clique no ícone de dispositivo móvel
3. Teste em diferentes tamanhos:
   - iPhone (375px)
   - iPad (768px)
   - Desktop (1920px)

No mobile:
- Menu lateral fica oculto
- Botão hambúrguer aparece
- Clique para abrir/fechar menu

═══════════════════════════════════════════════════════════════════
🐛 SOLUÇÃO DE PROBLEMAS
═══════════════════════════════════════════════════════════════════

PROBLEMA: Página em branco
SOLUÇÃO:
- Abra o Console (F12)
- Verifique erros em vermelho
- Confirme que todos os arquivos CSS/JS estão na mesma pasta

PROBLEMA: Estilos não carregam
SOLUÇÃO:
- Verifique se os arquivos existem:
  ✓ styles.css
  ✓ auth-styles.css
  ✓ dashboard-styles.css
- Todos devem estar na mesma pasta do HTML

PROBLEMA: Redireciona para login imediatamente
SOLUÇÃO:
- Isso é NORMAL! O sistema verifica autenticação
- Faça login primeiro em login.html
- Modo simulação aceita qualquer e-mail/senha

PROBLEMA: Logo não aparece
SOLUÇÃO:
- Substitua "logo-placeholder.png" por sua logo real
- Ou use uma logo temporária online

PROBLEMA: Fotos da galeria não aparecem
SOLUÇÃO:
- As imagens "portfolio-X-placeholder.jpg" devem existir
- Ou adicione suas próprias fotos

🔐 SOBRE O MODO SIMULAÇÃO
═══════════════════════════════════════════════════════════════════

⚠️ IMPORTANTE:
O sistema atual NÃO salva dados reais!

Modo Simulação significa:
✓ Qualquer e-mail/senha funcionam no login
✓ Não verifica se e-mail já existe no cadastro
✓ Não envia e-mails de recuperação
✓ Dados não persistem (somem ao recarregar)
✓ Não há validação server-side

Para PRODUÇÃO você precisa:
1. Configurar Firebase OU
2. Criar backend próprio OU
3. Usar Supabase

Veja o arquivo: GUIA_AUTENTICACAO.md
